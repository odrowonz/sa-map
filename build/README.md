# Каталог сборки и дистрибутива

**Исходный код здесь не хранится.** Исходники — в **`src/`** (в корне этого репозитория), когда появятся.

Сюда попадает только **артефакт сборки** — папка **`dist/`** (итог: PHP + статика + `vendor/` и т.д.), готовый к загрузке на хостинг в подкаталог (например `httpdocs/sa-map/`).

## Рекомендуемый процесс (после появления исходников)

1. Исходный код — в **`../src/`** (этот репозиторий) или в отдельном репозитории приложения.
2. Сборка локально или в CI из `src/`: `composer install --no-dev`, сборка фронтенда (`npm run build`), копирование артефактов в **`build/dist/`**.
3. В **`build/dist/`** должен лежать только то, что нужно на сервере (без `.env` с секретами — на сервере создать вручную).
4. Загрузка на хостинг по инструкции [../docs/HOSTING_INSTALL.md](../docs/HOSTING_INSTALL.md).

*Пока исходников продакшн-приложения нет — каталог пустой или содержит только этот README.*
